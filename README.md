# Dashboard 기능

현재 `feature/dashboard` 브랜치에서는 Streamlit 기반 대시보드 페이지를 구현했습니다.

대시보드는 로그인 상태의 사용자만 접근할 수 있으며, 고객 이탈 예측 결과와 운영 지표, 모델 진단 정보를 확인하는 화면으로 구성되어 있습니다.

## 구동 방식

대시보드는 Streamlit 멀티 페이지 구조로 동작합니다.

기본 진입점은 `app.py`이며, 사이드바 또는 랜딩 페이지의 **대시보드 보기** 버튼을 통해 대시보드 페이지로 이동할 수 있습니다.

로그인하지 않은 상태에서 대시보드에 접근하면 로그인 페이지로 이동하며, 로그인 이후 대시보드 화면을 확인할 수 있습니다.

## 대시보드 구성

대시보드는 크게 다음 3개 탭으로 구성되어 있습니다.

### 1. 개인

특정 `user_id`를 기준으로 고객 이탈 진단 정보를 조회합니다.

확인 가능한 항목은 다음과 같습니다.

- 최신 이탈 확률
- 위험 등급
- 예측 기준 기간
- 권장 액션
- 관심 브랜드
- 적용 모델
- 개인화 추천 카테고리

### 2. 운영

전체 서비스 운영 관점의 요약 정보를 확인합니다.

확인 가능한 항목은 다음과 같습니다.

- Best 모델
- Best ROC-AUC
- 모델별 성능 비교 데이터
- 고위험 고객 수
- 고위험 고객 목록
- 고객별 권장 액션

### 3. 모델 진단

모델별 성능 지표를 차트로 확인할 수 있습니다.

사용자는 진단 대상 모델을 선택하고, 확인할 지표 차트를 선택할 수 있습니다.

기본적으로 다음과 같은 지표 확인을 지원합니다.

- ROC-AUC
- PR-AUC
- Confusion Matrix

## 구동 확인 방법

1. 브라우저에서 Streamlit 앱이 실행되는지 확인합니다.
2. 랜딩 페이지에서 **대시보드 보기** 버튼을 클릭합니다.
3. 로그인되지 않은 상태라면 로그인 페이지로 이동하는지 확인합니다.
4. 로그인 또는 테스트 로그인 후 대시보드 페이지로 진입되는지 확인합니다.
5. 대시보드에서 다음 탭이 정상적으로 표시되는지 확인합니다.
   - 개인
   - 운영
   - 모델 진단
6. `user_id`를 입력했을 때 개인 이탈 진단 정보가 표시되는지 확인합니다.
7. 운영 탭에서 모델 요약 정보와 고위험 고객 목록이 표시되는지 확인합니다.
8. 모델 진단 탭에서 모델 선택 및 차트 선택 시 결과가 표시되는지 확인합니다.

## 참고

대시보드 데이터는 백엔드 API 응답을 기반으로 표시됩니다.  
따라서 실제 데이터 확인을 위해서는 API 서버가 실행 중이어야 하며, `.env`에 설정된 API 주소가 올바르게 연결되어 있어야 합니다.
